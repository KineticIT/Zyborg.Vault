New-VltAuth

NAME

New-VltAuth

SYNOPSIS

Authenticate with Vault against an auth provider using argument auth data.

Comparable to the vault CLI auth command.

SYNTAX

New-Auth [-AuthData <Hashtable>] [-AuthMethod <string>] [-KeepSecretWrapper <SwitchParameter>] [-MountName <string>] [-NoVerify <SwitchParameter>] [-PathData <string>] [-VaultAddress <string>] [-VaultProfile <string>] [-VaultSession <object>] [-VaultToken <string>] [<CommonParameters>]

New-Auth [-AuthData <Hashtable>] [-AuthMethod <string>] [-Force <SwitchParameter>] [-MountName <string>] [-NoVerify <SwitchParameter>] [-PathData <string>] [-SaveAsProfile <string>] [-VaultAddress <string>] [-VaultProfile <string>] [-VaultSession <object>] [-VaultToken <string>] [<CommonParameters>]

DESCRIPTION

By default, the AuthMethod is assumed to be token. If not supplied via the AuthData parameter, a prompt will be issued to collect the token ID used to authenticate against the token auth provider.

PARAMETERS

Selects the authentication method to be used, such as userpass, GitHub, or TLS certificates.

Each method corresponds to an associated auth provider which defines its own set of authentication parameters which are provided via the AuthData parameter.

EXAMPLES


---------- EXAMPLE 1 ----------

An example of authenticating to the 'userpass' auth provider at the default mount point.##

$creds = Get-Credential ## Interactively and securely prompt for auth creds
New-VltAuth -AuthMethod userpass -PathData $creds.Username ` 
          -AuthData @{ password = $creds.GetNetworkCredential().Password }

---------- EXAMPLE 2 ----------

An example of authenticating to the okta auth provider at the non-default mount point 'okta1'.##

$creds = Get-Credential ## Interactively and securely prompt for auth creds
New-VltAuth -AuthMethod okta -MountName okta1 -PathData $creds.Username ` 
          -AuthData @{ password = $creds.GetNetworkCredential().Password }
